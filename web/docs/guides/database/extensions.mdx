SQLコードが`create extension`（拡張を作成）していても、これは「拡張を有効にする」と同じことです。
拡張を無効にするには、`drop extension`を呼びだします。

SQLコード、`create extension <extension-name> with schema extensions`を実行した際、パーミッションの問題を引き起こすことがあります。例えば、`dblink`, `http`, `pg_cron`などがあります。
=======
Even though the SQL code is `create extension`, this is the equivalent of "enabling the extension". 
To disable an extension call `drop extension`.

:::caution
Enabling some extensions with `create extension <extension-name> with schema extensions` may lead to permission issues (e.g., `dblink`, `http`, `pg_cron`).
:::
